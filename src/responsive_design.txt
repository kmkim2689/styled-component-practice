* 화면의 크기에 상관없이 웹 페이지를 제대로 보여줄 수 있는 방법 : responsive design

--------------------------------------

* CSS Units
i) absolute Units
=> 기초적으로 써오던 것
항상 같은 사이즈로 보여지는 단위
px가 가장 대표적으로 사용
그 외 : pt, cm, mm, in 등

-------------------------------------

ii) relative units***
종류(units 종류)
1. relative to font-size***
- em
- rem

2. relative to the viewport
* viewport : browser window를 의미
- vw
- vh
- vmin
- vmax

-----------------------------------

iii) percentage
- 주로 너비(width)에 많이 사용된다
=> 높이에 사용하기에는 약간 이상함
- absolute units와 마찬가지로, 이해하기는 쉽다(the percentage that is relative to the parent)
- 100% 토탈에 맞게 비율만 설정하면 되기 때문에 상당히 계산하기 쉽다

화면 크기에 상관없이 잘 보여줄 수 있는 페이지를 만들기 위해, px만으로는 부족하다. 왜 px만으로는 안되는 것이며 percentage를 사용해야 하는 이유는 무엇인가

만약 화면의 전체 너비가 960px라고 한다면, 왼쪽의 그림 부분과 오른쪽의 로그인 부분으로 화면을 나누고자 한다. 왼쪽의 부분은 70%의 너비를 차지하도록 하고 싶고, 오른쪽 부분은 30%를 차지하도록 하고 싶다. 960px의 70%는 672px이 된다. 이렇게 너비를 설정하면 된다고 생각하면 곤란한데, 그 이유는 다른 요소들도 고려해야 하기 때문이다. width만 조절하면 안되고, 화면 크기에 따라 padding 등 다른 속성들도 조절해야 할 것인데, 이것은 매우 성가신 일이다. 그 이유는 px를 이용하여 fixed width를 설정하였기 때문이다...

특정 element의 width를 고정된 크기(예 : 500px)로 설정한다면, 만약 화면의 가로 크기를 줄였을 때 자동으로 요소의 너비가 줄어들지 않고 그대로 유지되며 화면 안에 온전히 보이지 않게 된다. 이러한 스크롤링 이슈가 발생하면 사용자 입장에서 웹페이지를 사용하기가 힘들어진다.

=> 따라서, width를 percentage 단위로 설정하여 이런 문제를 해결할 수 있다. 만약 width를 50%로 설정한다면, 화면의 크기를 줄이면 요소의 크기도 같이 줄어든다. 

+ 그런데, 해당 percentage 단위는 부모 요소의 높이/너비를 기준으로 적용되는 것임을 알야야 한다. 기본적으로 블록 레벨인 부모 요소의 너비 혹은 높이를 아무것도 설정하지 않으면 '기본적으로 값이 100%'로 설정된다. 즉, 부모 요소의 너비는 기본적으로 viewport를 꽉 채우게 된다는 의미이다.

만약 부모 요소의 너비를 특정 절대 픽셀값(예 : 500px)으로 설정한다면, 자식 요소의 너비를 50%로 설정한다면 자식요소의 너비는 '고정적으로' 250px이 될 것이다. 

*** 특정 element 내부의 이미지 너비를 관리하는 방법

특정 컨테이너(너비는 부모 요소의 50%) 내부에 이미지를 삽입할 때, 컨테이너 바깥으로 이미지가 빠져나오는 현상이 자주 발생하곤 한다. 

이것을 어떻게 해결하는가? 어떻게 부모 요소의 너비에 맞게 이미지의 너비를 설정할 수 있는가?
=> 부모 요소 내부에 들어간 자식 요소인 이미지의 너비를 percentage를 100%로 설정하면 그만이다.

그런데, image 태그는 'inline element'이다. 따라서, 만약 문자열로 이뤄진 문단 중간에 이미지를 삽입한다면, 글 중간에 이미지가 들어갈 것이다. 

(+ 이미지 크기 설정 시 주의할 점 : width 혹은 height 중 하나만 설정할 것. 하나만 구체적으로 설정하고 다른 하나는 auto로 설정하면 자동적으로 다른 것이 원본 비율에 따라 설정된다. 둘 다 수치를 설정하게 되면, 정교하지 않는 이상 왜곡이 일어나게 됨)

문제점 : 이미지의 너비는 적절하게 설정되었으나, 화면의 크기를 계속 줄인다면 이미지를 볼 수 없는 수준으로 너비가 짧아진다. 그 이유로 부모 요소(container)의 너비를 50%에서 90%로 늘린다고 해도, 화면이 클 때 문단의 한 줄 당 글자 수가 너무 많아져 가독성이 떨어지는 문제가 발생하게 된다.

* 이를 해결하는 방법이 바로 min-width와 max-width이다.
컨테이너 element의 width를 percentage로 설정하고, max-width를 절대수치(px)로 설정하게 된다면, 특정 너비 시점에서부터 더이상 너비가 늘어나지 않게 된다.
마찬가지로, min-width를 설정하면 화면을 줄여도 어느 시점부터는 너비가 줄어들지 않게 된다.
